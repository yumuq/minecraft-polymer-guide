## T1: 菜单类型注册

### 适用场景
- 模组有自定义 GUI 菜单
- 不需要方块/物品伪装
- 最简单的 Polymer 适配

### 代码模板

**文件：** `src/main/java/com/example/mod/ModMain.java`

```java
package com.example.mod;

import eu.pb4.polymer.core.api.other.PolymerMenuUtils;
import net.fabricmc.api.ModInitializer;

public class ModMain implements ModInitializer {
    
    @Override
    public void onInitialize() {
        // 现有的初始化代码
        ModMenuTypes.registerMenuTypes();
        
        // Polymer 适配：注册自定义菜单类型
        // 防止菜单类型同步到客户端，避免 RemapException
        PolymerMenuUtils.registerType(ModMenuTypes.CUSTOM_MENU_TYPE);
        
        // 如果有多个菜单类型，逐个注册
        // PolymerMenuUtils.registerType(ModMenuTypes.ANOTHER_MENU);
    }
}
```

### 关键点

1. **导入正确的类：**
   ```java
   import eu.pb4.polymer.core.api.other.PolymerMenuUtils;
   ```

2. **在菜单类型注册之后调用：**
   ```java
   MenuTypes.registerMenuTypes();  // 必须先注册
   PolymerMenuUtils.registerType(...);  // 然后标记为 Polymer
   ```

3. **所有自定义菜单都要注册：**
   ```java
   // 查找所有 MenuType 注册
   grep -r "Registry.register.*MENU" src/
   // 每个都要用 PolymerMenuUtils.registerType()
   ```

## T2: PolymerBlock 实现（Mixin 方式）

### 适用场景
- 有自定义方块
- 不能直接修改方块类源码
- 需要伪装为原版方块

### 完整模板

#### 步骤 1：创建 Mixin 配置文件

**文件：** `src/main/resources/modid-polymer.mixins.json`

```json
  "compatibilityLevel": "JAVA_25",

  ]

```

#### 步骤 2：创建 Mixin 类

**文件：** `src/main/java/com/example/mod/mixin/polymer/CustomBlockPolymerMixin.java`

```java
package com.example.mod.mixin.polymer;

import com.example.mod.block.CustomBlock;
import eu.pb4.polymer.core.api.block.PolymerBlock;
import io.github.fabricators_of_create.porting_lib.mixin.common.accessor.BlockBehaviourAccessor;
import net.minecraft.core.BlockPos;
import net.minecraft.server.level.ServerPlayer;
import net.minecraft.world.level.block.Block;
import net.minecraft.world.level.block.Blocks;
import net.minecraft.world.level.block.state.BlockState;
import net.minecraft.world.level.block.state.properties.BlockStateProperties;
import org.jetbrains.annotations.Nullable;
import org.spongepowered.asm.mixin.Mixin;
import org.spongepowered.asm.mixin.Unique;
import org.spongepowered.asm.mixin.injection.At;
import org.spongepowered.asm.mixin.injection.Inject;
import org.spongepowered.asm.mixin.injection.callback.CallbackInfo;

/**
 * Polymer 适配：让 CustomBlock 实现 PolymerBlock 接口
 * 使用 Mixin 软实现，不修改原始类
 */
@Mixin(CustomBlock.class)
@Implements(@Interface(iface = PolymerBlock.class, prefix = "polymer$"))
public abstract class CustomBlockPolymerMixin {
    
    /**
     * 存储伪装方块，避免重复创建
     */
    @Unique
    private BlockState polymer$polymerBlockState;
    
    /**
     * 构造函数注入：初始化伪装方块
     */
    @Inject(method = "<init>", at = @At("TAIL"))
    private void initPolymerBlock(
        BlockBehaviour.Properties properties,
        CallbackInfo ci
    ) {
        // 选择原版等效方块
        // 根据实际需求选择合适的原版方块
        this.polymer$polymerBlockState = Blocks.ANVIL.defaultBlockState();
        
        // 如果需要保持方块属性（如朝向），在 polymer$getPolymerBlockState 中处理
    }
    
    /**
     * PolymerBlock 接口方法：返回客户端看到的方块状态
     * 
     * @param state 服务器端真实的方块状态
     * @param context 数据包上下文（包含玩家信息）
     * @return 客户端应该看到的原版方块状态
     */
    public BlockState polymer$getPolymerBlockState(
        BlockState state, 
        PacketContext context
    ) {
        BlockState vanillaState = this.polymer$polymerBlockState;
        
        // 保留朝向属性
        if (vanillaState.hasProperty(BlockStateProperties.FACING) 
            && state.hasProperty(BlockStateProperties.FACING)) {
            vanillaState = vanillaState.setValue(
                BlockStateProperties.FACING,
                state.getValue(BlockStateProperties.FACING)
            );
        }
        
        // 保留损坏状态
        if (vanillaState.hasProperty(AnvilBlock.DAMAGE)
            && state.hasProperty(BlockStateProperties.LEVEL_ANVIL)) {
            vanillaState = vanillaState.setValue(
                AnvilBlock.DAMAGE,
                AnvilBlock.Damage.values()[
                    state.getValue(BlockStateProperties.LEVEL_ANVIL)
                ]
            );
        }
        
        // 可以根据玩家定制不同的伪装
        // ServerPlayer player = context.getPlayer();
        // if (player != null && player.hasPermission("see_real_block")) {
        //     return state;  // 特权玩家看到真实方块
        // }
        
        return vanillaState;
    }
    
    /**
     * 可选：自定义方块破坏音效
     */
    public Block polymer$getPolymerBlock(BlockState state, PacketContext context) {
        return this.polymer$polymerBlockState.getBlock();
    }
}
```

### 属性映射参考

```java
// 常见属性映射
// 朝向
BlockStateProperties.FACING → 大部分方块都有
BlockStateProperties.HORIZONTAL_FACING → 仅水平朝向

// 铁砧损坏
BlockStateProperties.LEVEL_ANVIL → AnvilBlock.DAMAGE

// 红石电源
BlockStateProperties.POWERED → 大部分红石元件

// 开关状态
BlockStateProperties.OPEN → 门、栅栏门

// 方向
BlockStateProperties.AXIS → 原木类方块
```

## T3: BlockEntity 注册

### 适用场景
- 方块有数据存储（BlockEntity）
- 需要同步 NBT 数据到客户端

### 代码模板

```java
import eu.pb4.polymer.core.api.block.PolymerBlockUtils;

@Override
public void onInitialize() {
    // 注册所有自定义 BlockEntity 类型
    PolymerBlockUtils.registerBlockEntity(
        ModBlockEntities.CUSTOM_BLOCK_ENTITY
    );
    
    // 如果有多个 BlockEntity
    PolymerBlockUtils.registerBlockEntity(
        ModBlockEntities.ANOTHER_BLOCK_ENTITY
    );
}
```

### 为什么需要注册？

```java
// 没有注册时：
服务器 BlockEntity NBT:
{
  id: "modid:custom_block_entity",
  items: [...],
  customData: {...}  // 客户端不认识
}
     ↓ 同步到客户端
客户端解析失败 → 数据丢失 → 渲染错误

// 注册后：
PolymerBlockUtils.registerBlockEntity(...);
     ↓ Polymer 自动过滤
客户端接收 NBT:
{
  id: "minecraft:chest",  // 伪装为原版
  items: [...]            // 保留兼容数据
  // customData 被过滤掉
}
```

## T4: PolymerItem 实现

### 适用场景
- 有自定义物品
- 需要伪装为原版物品

### 方式 A：直接实现（可以修改物品类）

```java
package com.example.mod.item;

import eu.pb4.polymer.core.api.item.PolymerItem;
import net.minecraft.server.level.ServerPlayer;
import net.minecraft.world.item.Item;
import net.minecraft.world.item.ItemStack;
import net.minecraft.world.item.Items;
import org.jetbrains.annotations.Nullable;

public class CustomItem extends Item implements PolymerItem {
    
    public CustomItem(Properties properties) {
        super(properties);
    }
    
    @Override
    public Item getPolymerItem(ItemStack itemStack, @Nullable ServerPlayer player) {
        // 客户端看到的物品
        return Items.DIAMOND_SWORD;
    }
}
```

### 方式 B：Mixin 实现（不能修改物品类）

```java
package com.example.mod.mixin.polymer;

import com.example.mod.item.CustomItem;
import eu.pb4.polymer.core.api.item.PolymerItem;
import net.minecraft.server.level.ServerPlayer;
import net.minecraft.world.item.Item;
import net.minecraft.world.item.ItemStack;
import net.minecraft.world.item.Items;
import org.jetbrains.annotations.Nullable;
import org.spongepowered.asm.mixin.Mixin;

@Mixin(CustomItem.class)
@Implements(@Interface(iface = PolymerItem.class, prefix = "polymer$"))
public abstract class CustomItemPolymerMixin {
    
    public Item polymer$getPolymerItem(
        ItemStack itemStack, 
        @Nullable ServerPlayer player
    ) {
        return Items.DIAMOND_SWORD;
    }
}
```

## T6: build.gradle 完整配置

### Fabric 单模块项目

```gradle
plugins {
    id 'fabric-loom' version '1.15-SNAPSHOT'
    id 'maven-publish'
}

version = project.mod_version
group = project.maven_group

repositories {
    mavenCentral()
    maven { url 'https://maven.fabricmc.net/' }
    
    // Polymer 仓库
    maven {
        name = "Nucleoid"
        url = "https://maven.nucleoid.xyz"
    }
}

dependencies {
    minecraft "com.mojang:minecraft:${project.minecraft_version}"
    mappings "net.fabricmc:yarn:${project.yarn_mappings}:v2"
    modImplementation "net.fabricmc:fabric-loader:${project.loader_version}"
    modImplementation "net.fabricmc.fabric-api:fabric-api:${project.fabric_version}"
    
    // Polymer Core - 版本号必须匹配 MC 版本
    implementation "eu.pb4:polymer-core:${project.polymer_version}"
    include "eu.pb4:polymer-core:${project.polymer_version}"
}

processResources {
    inputs.property "version", project.version
    filesMatching("fabric.mod.json") {
        expand "version": project.version
    }
}

tasks.withType(JavaCompile).configureEach {
    it.options.release = 25
}

java {
    withSourcesJar()
    sourceCompatibility = JavaVersion.VERSION_25
    targetCompatibility = JavaVersion.VERSION_25
}

jar {
    from("LICENSE") {
        rename { "${it}_${project.archivesBaseName}"}
    }
}
```

### gradle.properties 配置

```properties
# Minecraft 版本
minecraft_version=26.1.2
yarn_mappings=26.1.2+build.1
loader_version=0.18.4

# Fabric API
fabric_version=0.144.3+26.1

# Polymer - 必须匹配 MC 版本
polymer_version=0.16.5+26.1.2

# 模组信息
mod_version=1.0.0+polymer
maven_group=com.example.mod
archives_base_name=modname-polymer
```

### 多模块项目（Common + Fabric）

**settings.gradle:**
```gradle
rootProject.name = 'ModName'
include 'Common'
include 'Fabric'
```

**Fabric/build.gradle.kts:**
```kotlin
plugins {
    id("fabric-loom")
}

repositories {
    maven("https://maven.nucleoid.xyz") {
        name = "Nucleoid"
    }
}

dependencies {
    implementation(project(":Common"))
    
    minecraft("com.mojang:minecraft:${property("minecraft_version")}")
    mappings(loom.layered {
        officialMojangMappings()
    })
    
    modImplementation("net.fabricmc:fabric-loader:${property("fabric_loader_version")}")
    modImplementation("net.fabricmc.fabric-api:fabric-api:${property("fabric_api_version")}")
    
    // Polymer
    implementation("eu.pb4:polymer-core:${property("polymer_version")}")
    include("eu.pb4:polymer-core:${property("polymer_version")}")
}
```
