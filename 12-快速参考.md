### PolymerBlock 接口

```java
import eu.pb4.polymer.core.api.block.PolymerBlock;

public class CustomBlock extends Block implements PolymerBlock {
    
    @Override
    public BlockState getPolymerBlockState(
        BlockState state,
        PacketContext context
    ) {
        // 返回客户端看到的原版方块状态
        return Blocks.STONE.defaultBlockState();
    }
    
    @Override
    public Block getPolymerBlock(
        BlockState state,
        PacketContext context
    ) {
        // 返回用于音效的原版方块
        return Blocks.STONE;
    }
}
```

## Mixin 语法速查

### 基础 Mixin

```java
@Mixin(TargetClass.class)
public class TargetClassMixin {
    
    // 在方法开头注入
    @Inject(method = "targetMethod", at = @At("HEAD"))
    private void onMethodStart(CallbackInfo ci) {
        // 代码
    }
    
    // 在方法结尾注入
    @Inject(method = "targetMethod", at = @At("TAIL"))
    private void onMethodEnd(CallbackInfo ci) {
        // 代码
    }
    
    // 在方法返回前注入
    @Inject(method = "targetMethod", at = @At("RETURN"))
    private void beforeReturn(CallbackInfo ci) {
        // 代码
    }
}
```

---

### 软实现（Polymer 专用）

```java
@Mixin(TargetClass.class)
@Implements(@Interface(
    iface = PolymerBlock.class,
    prefix = "polymer$"
))
public abstract class TargetClassPolymerMixin {
    
    // 方法名 = 前缀 + 接口方法名
    public BlockState polymer$getPolymerBlockState(
        BlockState state,
        PacketContext context
    ) {
        return Blocks.STONE.defaultBlockState();
    }
}
```

---

### 构造函数注入

```java
@Mixin(TargetClass.class)
public class TargetClassMixin {
    
    @Unique
    private Object myField;
    
    @Inject(method = "<init>", at = @At("TAIL"))
    private void onConstruct(CallbackInfo ci) {
        this.myField = new Object();
    }
}
```

---

### 访问器

```java
@Mixin(TargetClass.class)
public interface TargetClassAccessor {
    
    @Accessor("privateField")
    Type getPrivateField();
    
    @Accessor("privateField")
    void setPrivateField(Type value);
    
    @Invoker("privateMethod")
    ReturnType invokePrivateMethod(Args args);
}
```

## 错误代码速查

| 错误关键词 | 类型 | 快速解决 |
|-----------|------|---------|
| `accesstransformer.cfg` | 构建 | `mkdir -p Common/build/generated/resources/META-INF && touch ...` |
| `Could not find polymer` | 依赖 | 添加 Nucleoid Maven 仓库 |
| `AbstractMethodError` | Mixin | 使用 `@Implements` 而非 `implements` |
| `RemapException` | 注册表 | `PolymerMenuUtils.registerType()` |
| `Mixin target not found` | Mixin | 检查类路径是否正确 |
| `NullPointerException` | 初始化 | 使用 `@Inject(method = "<init>")` |
| `ClassCastException` | 类型 | 使用 `instanceof` 检查 |

## 常见问题 FAQ

### Q: 如何判断模组类型？

**A:** 三步判断法：
1. 查看注册内容（方块/物品/菜单）
2. 查看交互方式（右键/快捷键）
3. 查看网络包（有/无）

---

### Q: 原版客户端为什么看到原版材质？

**A:** Polymer 将自定义方块伪装为原版方块，客户端只接收原版方块 ID，所以显示原版材质。功能由服务器端处理，不受影响。

---

### Q: 需要修改多少代码？

**A:** 取决于模组类型：
- 仅菜单：1 行
- 方块伪装：50-100 行
- 复杂系统：100+ 行

---

### Q: Polymer 会降低性能吗？

**A:** 影响极小：
- 注册表隔离：一次性操作
- 方块伪装：每次发包时转换（微秒级）
- 实际游戏中几乎无法察觉

---

### Q: 能否让原版客户端看到自定义材质？

**A:** 不能。Polymer 只能让原版客户端**连接**和使用**功能**，不能让原版客户端显示自定义内容（因为客户端没有资源文件）。

---

### Q: 适配后需要维护吗？

**A:** 很少：
- Minecraft 版本更新 → 更新 Polymer 版本
- 模组添加新注册项 → 添加对应注册
- 通常只需几分钟
