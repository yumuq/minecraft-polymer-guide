## 构建阶段错误

### E1: accesstransformer.cfg 不存在

**错误信息：**
```
FileNotFoundException: Common/build/generated/resources/META-INF/accesstransformer.cfg
```

**原因：**
NeoForge Gradle 插件要求此文件存在，但项目没有生成。

**解决方案：**
```bash
# 方案 1：创建空文件
mkdir -p Common/build/generated/resources/META-INF
touch Common/build/generated/resources/META-INF/accesstransformer.cfg

# 方案 2：禁用 NeoForge 插件（如果不需要）
# 在 build.gradle 中注释掉相关配置

# 重新构建
./gradlew clean build
```

**预防措施：**
在 `.gitignore` 中添加：
```gitignore
**/build/generated/
```

### E2: 找不到 Polymer 依赖

**错误信息：**
```
Could not find eu.pb4:polymer-core:0.16.5+26.1.2
```

**原因：**
1. Nucleoid Maven 仓库未添加
2. Polymer 版本号错误
3. 网络连接问题

**解决方案：**

**步骤 1：检查仓库配置**
```gradle
repositories {
    maven {
        name = "Nucleoid"
        url = "https://maven.nucleoid.xyz"
    }
}
```

**步骤 2：验证版本号**
访问：https://maven.nucleoid.xyz/eu/pb4/polymer-core/
```
目录列表：
- 0.16.5+26.1.2/  ← 匹配 MC 26.1.2
- 0.15.0+1.21.1/
- ...
```

**步骤 3：检查网络**
```bash
curl -I https://maven.nucleoid.xyz/eu/pb4/polymer-core/
# 应返回 200 OK
```

**步骤 4：清除缓存**
```bash
./gradlew clean --refresh-dependencies
```

### E3: ModMenu 依赖不可用

**错误信息：**
```
Could not find com.terraformersmc:modmenu:18.0.0-alpha.8
```

**原因：**
ModMenu 的 alpha 版本可能未发布或仓库不可用。

**解决方案：**

**步骤 1：注释依赖**
```gradle
dependencies {
    // implementation("com.terraformersmc:modmenu:18.0.0-alpha.8")
}
```

**步骤 2：删除集成代码**
```bash
rm src/main/java/.../ModMenuIntegration.java
```

**步骤 3：清理 fabric.mod.json**
```json
{
  "entrypoints": {
    "main": [...],
    // 删除 modmenu 入口点
    // "modmenu": ["com.example.ModMenuIntegration"]
  }
}
```

**步骤 4：清理语言文件**
删除 `assets/modid/lang/*.json` 中的 `modmenu.*` 翻译项。

### E4: 编译错误 - 找不到 PolymerBlock

**错误信息：**
```
cannot find symbol: class PolymerBlock
```

**原因：**
1. Polymer 依赖未正确添加
2. `include` 语句缺失
3. IDE 未刷新依赖

**解决方案：**

```gradle
dependencies {
    // 两行都需要！
    implementation "eu.pb4:polymer-core:0.16.5+26.1.2"
    include "eu.pb4:polymer-core:0.16.5+26.1.2"  // ← 必须
}
```

刷新 IDE：
```bash
./gradlew clean
./gradlew idea  # IntelliJ IDEA
./gradlew eclipse  # Eclipse
```

### E5: Mixin 编译失败

**错误信息：**
```
Mixin target com.example.mod.block.CustomBlock was not found
```

**原因：**
1. Mixin 目标类路径错误
2. 目标类不存在
3. Mixin 配置未加载

**解决方案：**

**检查目标类路径：**
```java
@Mixin(CustomBlock.class)  // ← 完整类名正确？
```

**验证类存在：**
```bash
find src/ -name "CustomBlock.java"
```

**检查 Mixin 配置加载：**
```json
// fabric.mod.json
{
  "mixins": [
    "modid.mixins.json",
    "modid-polymer.mixins.json"  // ← 添加了吗？
  ]
}
```

## 启动阶段错误

### E6: Mixin 应用失败

**错误信息：**
```
Mixin apply failed: modid-polymer.mixins.json:CustomBlockPolymerMixin
```

**原因：**
1. Mixin 语法错误
2. 目标方法签名不匹配
3. 注入点错误

**解决方案：**

**检查接口实现语法：**
```java
// ❌ 错误
@Mixin(CustomBlock.class)
public abstract class Mixin implements PolymerBlock {
    // 这不会修改目标类！
}

// ✅ 正确
@Mixin(CustomBlock.class)
@Implements(@Interface(iface = PolymerBlock.class, prefix = "polymer$"))
public abstract class Mixin {
    public BlockState polymer$getPolymerBlockState(...) {
        // 这会注入到目标类
    }
}
```

**检查方法签名：**
```java
// 确保参数完全匹配
public BlockState polymer$getPolymerBlockState(
    BlockState state,           // ← 顺序和类型
    PacketContext context       // ← 必须正确
)
```

### E7: AbstractMethodError

**错误信息：**
```
java.lang.AbstractMethodError: 
  Receiver class com.example.mod.block.CustomBlock 
  does not define or inherit an implementation of the resolved method 
  'abstract BlockState getPolymerBlockState(BlockState, PacketContext)'
```

**原因：**
Mixin 软实现失败，接口方法未被注入到目标类。

**根本原因：**
1. 使用了 `implements` 而非 `@Implements`
2. 方法前缀错误
3. 方法签名不匹配

**解决方案：**

**正确的 Mixin 实现：**
```java
@Mixin(CustomBlock.class)
@Implements(@Interface(
    iface = PolymerBlock.class,
    prefix = "polymer$"  // ← 必须有前缀
))
public abstract class CustomBlockPolymerMixin {
    
    // 方法名 = prefix$ + 接口方法名
    public BlockState polymer$getPolymerBlockState(
        BlockState state,
        PacketContext context
    ) {
        return Blocks.STONE.defaultBlockState();
    }
}
```

**验证方法：**
```bash
# 构建后检查字节码
javap -c -cp build/libs/*.jar com.example.mod.block.CustomBlock

# 应该看到 getPolymerBlockState 方法
```

## 连接阶段错误

### E8: RemapException - Registry entry missing

**错误信息：**
```
net.minecraft.network.protocol.common.custom.GamePackets$Exception: 
  RemapException: Registry entry missing for 'modid:custom_menu'
```

**原因：**
自定义菜单类型未使用 Polymer 注册，尝试同步到客户端。

**解决方案：**

```java
import eu.pb4.polymer.core.api.other.PolymerMenuUtils;

@Override
public void onInitialize() {
    // 在注册菜单类型之后
    ModMenuTypes.registerMenuTypes();
    
    // 添加 Polymer 注册
    PolymerMenuUtils.registerType(ModMenuTypes.CUSTOM_MENU);
}
```

**验证所有菜单：**
```bash
# 查找所有菜单类型注册
grep -r "Registry.register.*MENU" src/

# 每个都需要 PolymerMenuUtils.registerType()
```

### E9: 客户端断开连接 - 无具体错误

- 模组客户端正常

**2. 检查客户端日志：**
客户端 `logs/latest.log` 中查找：
- `RemapException`
- `Registry`
- `Disconnected`

**3. 逐步排除：**
```java
// 临时注释所有 Polymer 注册
// PolymerMenuUtils.registerType(...);
// PolymerBlockUtils.registerBlockEntity(...);

// 逐个启用，找出问题项
```

### E10: 注册表 ID 不匹配

**错误信息：**
```
Registry remapping failed: 
  Expected 'modid:custom_block' but got 'minecraft:stone'
```

**原因：**
Polymer 伪装配置错误，客户端预期与实际不符。

**解决方案：**

检查 `getPolymerBlockState` 返回值：
```java
public BlockState polymer$getPolymerBlockState(...) {
    // 必须返回真实的原版方块状态
    return Blocks.STONE.defaultBlockState();  // ✅
    
    // 不能返回 null 或自定义方块
    // return null;  // ❌
    // return ModBlocks.CUSTOM.defaultBlockState();  // ❌
}
```

## 运行阶段错误

### E11: NullPointerException 在 getPolymerBlockState

**错误信息：**
```
java.lang.NullPointerException
  at CustomBlockPolymerMixin.polymer$getPolymerBlockState
```

**原因：**
访问了未初始化的字段或方法。

**解决方案：**

**使用构造函数注入初始化：**
```java
@Mixin(CustomBlock.class)
@Implements(@Interface(iface = PolymerBlock.class, prefix = "polymer$"))
public abstract class CustomBlockPolymerMixin {
    
    @Unique
    private BlockState polymer$cachedState;
    
    @Inject(method = "<init>", at = @At("TAIL"))
    private void initPolymer(CallbackInfo ci) {
        // 初始化伪装方块
        this.polymer$cachedState = Blocks.ANVIL.defaultBlockState();
    }
    
    public BlockState polymer$getPolymerBlockState(...) {
        // 现在 polymer$cachedState 不会是 null
        return polymer$cachedState;
    }
}
```

### E12: ClassCastException

**错误信息：**
```
java.lang.ClassCastException: 
  net.minecraft.world.level.block.Block cannot be cast to 
  eu.pb4.polymer.core.api.block.PolymerBlock
```

**原因：**
尝试将不是 PolymerBlock 的方块强制转换。

**解决方案：**

**使用 instanceof 检查：**
```java
if (block instanceof PolymerBlock polymerBlock) {
    BlockState polymerState = polymerBlock.getPolymerBlockState(...);
    // 安全使用
} else {
    // 不是 Polymer 方块
}
```

### E13: 方块属性丢失

**现象：**
- 客户端看到的方块朝向错误
- 方块状态不正确

**原因：**
`getPolymerBlockState` 没有复制属性。

**解决方案：**

```java
public BlockState polymer$getPolymerBlockState(
    BlockState state,
    PacketContext context
) {
    BlockState vanilla = Blocks.ANVIL.defaultBlockState();
    
    // 复制所有共同属性
    for (Property<?> property : state.getProperties()) {
        if (vanilla.hasProperty(property)) {
            vanilla = copyProperty(state, vanilla, property);
        }
    }
    
    return vanilla;
}

@SuppressWarnings("unchecked")
private static <T extends Comparable<T>> BlockState copyProperty(
    BlockState source,
    BlockState target,
    Property<T> property
) {
    return target.setValue(
        (Property<T>) property,
        source.getValue(property)
    );
}
```

### E14: BlockEntity 数据未同步

**现象：**
- 客户端看不到容器内物品
- 自定义数据丢失

**原因：**
BlockEntity 未注册到 Polymer。

**解决方案：**

```java
import eu.pb4.polymer.core.api.block.PolymerBlockUtils;

@Override
public void onInitialize() {
    PolymerBlockUtils.registerBlockEntity(
        ModBlockEntities.CUSTOM_BLOCK_ENTITY
    );
}
```

**验证：**
```java
// 在 BlockEntity 类中添加日志
@Override
public void saveAdditional(CompoundTag tag, HolderLookup.Provider registries) {
    super.saveAdditional(tag, registries);
    System.out.println("Saving BlockEntity data: " + tag);
}
```

## 错误排查流程

### 通用排查步骤

```
1. 确定错误阶段
   ↓
2. 查看完整错误栈
   ↓
3. 定位错误来源
   ├─ 构建错误 → 检查 build.gradle
   ├─ Mixin 错误 → 检查 Mixin 配置
   ├─ 连接错误 → 检查注册代码
   └─ 运行错误 → 检查逻辑实现
   ↓
4. 应用解决方案
   ↓
5. 验证修复
```

---

**1. 启用详细日志：**
```bash
# 启动参数
-Dmixin.debug.verbose=true
```

## 错误速查表

| 错误信息关键词 | 可能原因 | 快速解决 |
|--------------|---------|---------|
| `accesstransformer.cfg` | NeoForge 插件 | 创建空文件 |
| `Could not find polymer-core` | 仓库未添加 | 添加 Nucleoid Maven |
| `Could not find modmenu` | 依赖不可用 | 注释依赖 |
| `AbstractMethodError` | Mixin 实现错误 | 使用 @Implements |
| `RemapException` | 注册表未隐藏 | PolymerMenuUtils.registerType() |
| `Mixin target not found` | 目标类路径错误 | 检查 @Mixin 参数 |
| `NullPointerException` | 未初始化 | 使用 @Inject 初始化 |
| `ClassCastException` | 类型转换错误 | 使用 instanceof |
