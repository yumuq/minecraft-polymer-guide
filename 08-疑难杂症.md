## 构建系统问题

### 多模块项目构建失败

**问题：** Common 模块找不到 Fabric API

**错误信息：**
```
Could not resolve net.fabricmc.fabric-api:fabric-api
```

**原因：** Common 模块不应直接依赖 Fabric 特定的 API

**解决方案：**

**Common/build.gradle.kts:**
```kotlin
dependencies {
    // 只依赖核心接口
    compileOnly("net.fabricmc:fabric-loader:${property("loader_version")}")
    
    // Polymer Core
    implementation("eu.pb4:polymer-core:${property("polymer_version")}")
}
```

**Fabric/build.gradle.kts:**
```kotlin
dependencies {
    // 包含 Common 模块
    implementation(project(":Common"))
    
    // Fabric 特定依赖
    modImplementation("net.fabricmc.fabric-api:fabric-api:${property("fabric_version")}")
    
    // 打包 Polymer
    include("eu.pb4:polymer-core:${property("polymer_version")}")
}
```

### Gradle 缓存损坏

**问题：** 构建失败，清理后仍然失败

**症状：**
```
Could not resolve dependencies
Cached resource is invalid
```

**解决方案：**

```bash
# 1. 删除 Gradle 缓存
rm -rf ~/.gradle/caches/

# Windows
rmdir /s /q %USERPROFILE%\.gradle\caches

# 2. 删除项目缓存
rm -rf .gradle/
rm -rf build/

# 3. 重新构建
./gradlew clean build --refresh-dependencies
```

### AccessTransformer 循环依赖

**问题：** NeoForge 和 Fabric 混合项目

**错误信息：**
```
Circular dependency between:
  - accessTransformer
  - generateResources
```

**解决方案：**

**方案 1：创建空文件**
```bash
mkdir -p Common/build/generated/resources/META-INF
touch Common/build/generated/resources/META-INF/accesstransformer.cfg
```

**方案 2：禁用 AccessTransformer（仅 Fabric）**

编辑 `Common/build.gradle.kts`:
```kotlin
tasks.named("processResources") {
    // 跳过 AccessTransformer
    exclude("META-INF/accesstransformer.cfg")
}
```

**方案 3：分离 Fabric 和 Forge 构建**
```
项目结构：
├── Common/
├── Fabric/
└── Forge/     # 完全独立的 Forge 构建
```

## 依赖冲突问题

### Mixin 版本冲突

**问题：** 多个模组使用不同版本的 Mixin

**错误信息：**
```
java.lang.NoSuchMethodError: org.spongepowered.asm.mixin.Mixin
```

**诊断：**
```bash
# 查看依赖树
./gradlew dependencies --configuration runtimeClasspath

# 查找 Mixin 版本
./gradlew dependencies | grep -i mixin
```

**解决方案：**

强制使用特定版本：
```gradle
configurations.all {
    resolutionStrategy {
        force 'org.spongepowered:mixin:0.8.5'
    }
}
```

### Fabric API 模块冲突

**问题：** 某些 Fabric API 模块重复

**错误信息：**
```
Duplicate class found in modules:
  fabric-api-base
  fabric-networking-api-v1
```

**解决方案：**

排除特定模块：
```gradle
dependencies {
    modImplementation("net.fabricmc.fabric-api:fabric-api:${fabric_version}") {
        exclude group: 'net.fabricmc.fabric-api', module: 'fabric-api-base'
    }
}
```

### Polymer 与其他库冲突

**问题：** Polymer 与其他虚拟化库冲突

**示例：** Polymer + ServerCore

**解决方案：**

1. **检查兼容性：** 查看两个库的文档
2. **分别测试：** 逐个添加依赖
3. **调整加载顺序：**

编辑 `fabric.mod.json`:
```json
{
  "depends": {
    "polymer-core": "*"
  },
  "breaks": {
    "conflicting-mod": "*"
  }
}
```

## Mixin 高级问题

### Mixin 优先级冲突

**问题：** 多个 Mixin 修改同一方法

**错误信息：**
```
Critical injection failure: 
Callback injection failed
```

**解决方案：**

设置 Mixin 优先级：

**modid.mixins.json:**
```json
{
  "required": true,
  "priority": 1000,
  "package": "com.example.mod.mixin",
  "mixins": ["CustomMixin"]
}
```

**优先级规则：**
- 数字越大，优先级越高
- 默认优先级：1000
- Polymer 推荐：900-1100

---

### Mixin 目标类找不到

**问题：** Mixin 在某些环境下找不到目标类

**错误信息：**
```
Mixin target com.example.TargetClass was not found
```

**可能原因：**
1. 目标类在某些模组配置下不加载
2. 目标类名拼写错误
3. 目标类在不同环境有不同包路径

**解决方案：**

**使用可选 Mixin：**
```json
{
  "required": false,
  "mixins": ["OptionalMixin"]
}
```

**条件加载：**
```java
@Mixin(value = TargetClass.class, remap = false)
@Pseudo  // 标记为伪类
public class ConditionalMixin {
    // Mixin 代码
}
```

### Mixin 与 Refmap 问题

**问题：** 混淆环境下 Mixin 失败

**错误信息：**
```
Failed to apply refmap
```

**解决方案：**

确保生成 Refmap：

**build.gradle:**
```gradle
loom {
    mixinConfig "modid.mixins.json"
    
    // 启用 Refmap 生成
    accessWidenerPath = file("src/main/resources/modid.accesswidener")
}
```

## 运行时性能问题

### Polymer 转换性能下降

**症状：** 大量方块更新时卡顿

**诊断：**
```java
@Override
public BlockState getPolymerBlockState(BlockState state, PacketContext context) {
    long start = System.nanoTime();
    BlockState result = Blocks.STONE.defaultBlockState();
    long duration = System.nanoTime() - start;
    if (duration > 1000000) {  // > 1ms
        System.err.println("Slow conversion: " + duration + "ns");
    }
    return result;
}
```

**优化方案：**

**1. 缓存伪装状态：**
```java
@Unique
private final BlockState[] polymer$cachedStates = new BlockState[16];

@Inject(method = "<init>", at = @At("TAIL"))
private void initCache(CallbackInfo ci) {
    // 预计算所有可能的状态
    for (int i = 0; i < 16; i++) {
        polymer$cachedStates[i] = computePolymerState(i);
    }
}

public BlockState polymer$getPolymerBlockState(BlockState state, PacketContext context) {
    int index = getStateIndex(state);
    return polymer$cachedStates[index];
}
```

**2. 避免重复计算：**
```java
// ❌ 慢
public BlockState polymer$getPolymerBlockState(...) {
    BlockState vanilla = Blocks.ANVIL.defaultBlockState();
    vanilla = vanilla.setValue(FACING, state.getValue(FACING));
    vanilla = vanilla.setValue(DAMAGE, computeDamage(state));
    return vanilla;
}

// ✅ 快
private static final Map<BlockState, BlockState> CACHE = new HashMap<>();

public BlockState polymer$getPolymerBlockState(...) {
    return CACHE.computeIfAbsent(state, s -> {
        BlockState vanilla = Blocks.ANVIL.defaultBlockState();
        // 计算...
        return vanilla;
    });
}
```

### BlockEntity 同步开销大

**症状：** 网络流量异常高

**诊断：**
```bash
# 启用网络调试
java -Dpolymer.debug=true -jar server.jar

# 查看日志
grep "BlockEntity sync" logs/latest.log
```

**优化方案：**

**1. 减少同步频率：**
```java
@Override
public void saveAdditional(CompoundTag tag, HolderLookup.Provider registries) {
    super.saveAdditional(tag, registries);
    
    // 只在需要时同步
    if (isDirty) {
        tag.put("items", items.serializeNBT());
        isDirty = false;
    }
}
```

**2. 压缩数据：**
```java
// 只发送必要的数据
@Override
public CompoundTag getUpdateTag(HolderLookup.Provider registries) {
    CompoundTag tag = new CompoundTag();
    // 只包含客户端需要的数据
    tag.putInt("displayCount", items.size());
    // 不发送完整的 items 列表
    return tag;
}
```

### 与数据包模组冲容

**问题：** 数据包修改方块属性后 Polymer 转换错误

**解决方案：**

动态适应方块属性：
```java
public BlockState polymer$getPolymerBlockState(BlockState state, PacketContext context) {
    BlockState vanilla = Blocks.ANVIL.defaultBlockState();
    
    // 动态检查可用属性
    for (Property<?> property : state.getProperties()) {
        if (vanilla.hasProperty(property)) {
            vanilla = copyProperty(state, vanilla, property);
        }
    }
    
    return vanilla;
}
```

## 生产环境问题

### 内存泄漏

**症状：** 服务器长时间运行后内存持续增长

**诊断：**
```bash
# 生成堆转储
jmap -dump:format=b,file=heap.hprof <pid>

# 分析
jhat heap.hprof
# 或使用 VisualVM, Eclipse MAT
```

**常见原因：**
1. Polymer 缓存未清理
2. 静态集合持续增长
3. 事件监听器未注销

**解决方案：**
```java
// 定期清理缓存
@Unique
private static final Map<BlockState, BlockState> CACHE = 
    Collections.synchronizedMap(new WeakHashMap<>());
```

cp backup/original-mod.jar mods/

# 4. 重启服务器
java -jar server.jar
