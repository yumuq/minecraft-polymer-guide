## 通用流程（5 步）

### Phase 1: 分析模组

```bash
# 1. 解压源码
unzip mod-source.zip -d mod-src/

# 2. 判断模组类型
grep -r "Registry.register" src/
grep -r "UseBlockCallback\|UseItemCallback" src/
grep -r "KeyBinding" src/

# 3. 查找自定义注册项
grep -r "BLOCK_ENTITY_TYPE\|MENU\|ITEM\|BLOCK" src/main/java/*/init/
```

**决策点：**
- 有自定义方块？ → 需要 PolymerBlock
- 有自定义菜单？ → 需要 PolymerMenuUtils
- 有 BlockEntity？ → 需要 PolymerBlockUtils

---

### Phase 2: 添加依赖

**编辑 build.gradle:**

**版本查找：**

- 找到对应 Minecraft 版本的 Polymer

### Phase 3: 修改代码

#### 场景 A：仅菜单注册（Quick Shulker 类型）

**文件：** `ModMain.java`

```java
import eu.pb4.polymer.core.api.other.PolymerMenuUtils;

@Override
public void onInitialize() {
    // 现有初始化代码...
    
    // 添加 Polymer 注册
    PolymerMenuUtils.registerType(ModMenuTypes.CUSTOM_MENU);
}
```

**时间：** 5 分钟  
**难度：** ⭐ 极简

---

#### 场景 B：方块伪装（Easy Anvils 类型）

**步骤 1：创建 Mixin 配置**

`resources/mod-polymer.mixins.json:`
```json
{
  "required": true,
  "package": "com.example.mod.mixin",
  "compatibilityLevel": "JAVA_21",
  "mixins": ["CustomBlockPolymerMixin"]
}
```

**步骤 2：实现 Mixin**

`mixin/CustomBlockPolymerMixin.java:`
```java
@Mixin(CustomBlock.class)
@Implements(@Interface(iface = PolymerBlock.class, prefix = "polymer$"))
public abstract class CustomBlockPolymerMixin {
    
    public BlockState polymer$getPolymerBlockState(
        BlockState state, 
        PacketContext context
    ) {
        // 返回原版等效方块
        BlockState vanilla = Blocks.STONE.defaultBlockState();
        
        // 复制属性（如朝向）
        if (vanilla.hasProperty(BlockStateProperties.FACING)) {
            vanilla = vanilla.setValue(
                BlockStateProperties.FACING,
                state.getValue(BlockStateProperties.FACING)
            );
        }
        
        return vanilla;
    }
}
```

**步骤 3：注册组件**

`ModMain.java:`
```java
import eu.pb4.polymer.core.api.block.PolymerBlockUtils;
import eu.pb4.polymer.core.api.other.PolymerMenuUtils;

@Override
public void onInitialize() {
    // BlockEntity 注册
    PolymerBlockUtils.registerBlockEntity(
        ModBlockEntities.CUSTOM_BLOCK_ENTITY
    );
    
    // 菜单类型注册
    PolymerMenuUtils.registerType(ModMenuTypes.CUSTOM_MENU);
}
```

**步骤 4：更新 fabric.mod.json**

手动或 Gradle 自动化：
```json
{
  "mixins": [
    "mod.mixins.json",
    "mod-polymer.mixins.json"
  ]
}
```

**时间：** 1-2 小时  
**难度：** ⭐⭐⭐ 中等

### Phase 4: 构建验证

```bash
# 清理构建
./gradlew clean build --no-daemon

# 常见问题快速修复
# 问题 1: accesstransformer.cfg 不存在
mkdir -p Common/build/generated/resources/META-INF
touch Common/build/generated/resources/META-INF/accesstransformer.cfg

# 问题 2: ModMenu 依赖不可用
# 注释掉 build.gradle 中的 modmenu 依赖
# 删除 ModMenuIntegration.java
# 清理 fabric.mod.json 中的 modmenu 入口点

# 验证 JAR
unzip -l build/libs/*.jar | grep polymer
# 应该看到 polymer-core.jar
```

## 错误速查

### 构建错误

| 错误信息 | 原因 | 解决方案 |
|---------|------|---------|
| `accesstransformer.cfg not found` | NeoForge 插件要求 | 创建空文件 |
| `Could not find modmenu` | 依赖不可用 | 注释依赖，删除集成代码 |
| `Method getPolymerBlockState is abstract` | 方法签名错误 | 检查参数：`(BlockState, PacketContext)` |

---

### 运行时错误

| 错误信息 | 原因 | 解决方案 |
|---------|------|---------|
| `RemapException: Registry entry missing` | 菜单类型未注册 | `PolymerMenuUtils.registerType()` |
| `AbstractMethodError` | Mixin 实现错误 | 使用 `@Implements` 而非 `implements` |
| 客户端断开连接 | 多种可能 | 查看服务器日志具体错误 |
